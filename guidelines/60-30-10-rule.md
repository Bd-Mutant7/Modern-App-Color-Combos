# ğŸ¯ The 60-30-10 Rule

## What is it?
A timeless interior design rule adapted for UI design to create balanced color schemes.

## Distribution
60% - Dominant Color (Backgrounds, large areas)
30% - Secondary Color (UI elements, cards)
10% - Accent Color (CTAs, notifications)


## Examples

### Health App
```css
:root {
  --dominant-60: #FFFFFF;  /* Background */
  --secondary-30: #4ECDC4; /* Cards, headers */
  --accent-10: #FF6B6B;    /* Buttons, alerts */
}
```
### Fintech App

```css
:root {
  --dominant-60: #0A2342;  /* Dark background */
  --secondary-30: #2C7A7B; /* Secondary elements */
  --accent-10: #D4AF37;    /* CTAs, premium badges */
}
```
### Why it works
- Creates visual hierarchy
- Prevents overwhelming users
- Guides attention naturally
- Maintains brand recognition

### 10. Create a validation script `scripts/validate-colors.js`
```javascript
// scripts/validate-colors.js
const fs = require('fs');
const path = require('path');
const ccc = require('color-contrast-checker');

const categories = [
  'health', 'fitness', 'fintech', 'ecommerce', 
  'education', 'navigation', 'entertainment', 'social'
];

function validatePalette(colors) {
  const results = {};
  
  // Check text contrast (assuming white background)
  if (colors.text && colors.background) {
    results.textContrast = ccc.verify(colors.text, colors.background, 'AA');
  }
  
  // Check button contrast
  if (colors.primary && colors.background) {
    results.buttonContrast = ccc.verify(colors.primary, colors.background, 'AA');
  }
  
  return results;
}

categories.forEach(category => {
  console.log(`Validating ${category}...`);
  // Read and validate each category's palettes
});

console.log('âœ… Validation complete!');
```
